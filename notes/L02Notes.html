Master Mobile UX

 

·      Design for the smallest and weakest screens first, then build up.

·      Mobile devices are selling a lot higher and need to be seen after when it comes to user design

·      Phones and tablets are replaced a lot faster than computers.

·      Responsive Retrofitting – Making a desktop site compatible for mobile

·      Mobile first is the most open to growth.

·      Be there. Be Quick.

·      Reduce steps and optimize images for slower networks and devices

·      Stop using pop up ads

 

JavaScript: Novice to Ninja Ch. 12

 

·      Object-oriented programming:

o   Encapsulation – Polymorphism – Inheritance

·      Constructor functions

·      .propertyIsEnumerable() – checks if they show up in a for-in loop

·      .getOwnPropertyDescriptor() – returns a list of properties

·      .defineProperty() – allows you to set new properties to an object

·      Mixins

·      Object.assign(a, b) – sets the properties of a to b

·      Chaining functions – object.act1().act2().act3() – performs all three actions in a row

·      this – references an object within one of its methods

·      when using this to iterate through a list:

o   set that = this first, then use .forEach()

o   use .bind(this) at the end to bind the attribute to the same class

Modern JavaScript

 

·      Libraries –

·      jQuery for DOM Manipulation

·      Underscore and Lodash

·      Libraries are useful but using too many means a lot of downloading for the website. They can also be slower and apply to less browsers than hand written code.

·      Repository of JS libraries or vanilla resources:

o   http://microjs.com/#

o   http://anguscroll.com/just/

o   http://vanilla-js.com

·      export/import – let you use functions from other .js files

·      export default – makes the single export simpler (cannot use more than 1)

·      MVC Frameworks – with JS, it looks more like MV (Models / Views)

·      Templates let you insert dynamic code into an html file from across files.

·      React / Vue.js – View libraries.

·      View libraries have a fake DOM in memory and only make real DOM changes at the end

·      Package managers like npm allow you to install packages and manage dependencies

·      Code transpiling lets you change js code for compatibility with older systems
