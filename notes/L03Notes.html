The Window Object

 

·      The browser window is an object representing the device screen

·      window is THE global object of a browser page.

·      pro tip: if you don’t know the name of the browser object set const global = this; and this will become the name of the global object

·      Any declared global object becomes the property of window

o   name = “ryan” makes window.name = “ryan”

·      alert(), confirm(), prompt() – accepted window methods

·      window.navigator.userAgent – finds properties of which browser is being used

·      window.location properties:

o   .href – returns exact link

o   .protocol – returns protocol (https, etc)

o   .host – returns url and port #

o   .hostname – returns url domain w/o port

o   .pathname – returns path after domain

o   .search – returns all search parameters from previous forms

o   .hash – returns fragment identifier

o   .origin – returns protocol + domain

o   .reload() – force reload page

o   .assign() – loads the given url

o   .replace() – same as assign but you can’t use back button

o   .toString() – returns whole url as string

·      window.history contains information about previously visited pages. Properties:

o   .length – shows how many pages have been visited before the site

o   .go() – travel the history (0 is the current page)

o   .forward(), .back() – moves through the history one site at a time

·      window.open() – opens a new window with these parameters:

o   link url, window name, css modifiers

o   can be set to a variable

·      var.close() – closes the referenced window

·      window.resizeTo() – sets two new parameters, width and height

·      window.screen – contains the following properties:

o   .height, width – height & width

o   .availHeight, availWidth – height & width excluding browser menus/bars

o   .colorDepth

·      The document object deals with the actual page loaded on the window.

·      document.write() – forces the desired text or element onto a page

·      pro tip: the write method represents a messy and disorganized javascript – don’t use

·      document.cookie – returns the current cookies as a string – try it on any page’s console!

·      document.cookie = ‘cookie=value’; – appends the given cookie to the string. If the cookie is already set, replaces it with the new value

 

example of cookie retrieval:

const cookies = document.cookie.split(“; “);

for (crumb of cookies){

     const [key,value] = crumb.split(“=”);

console.log(key, value);

}

 

·      cookie ‘expires=’ – lets you set expiration date as ‘date’ format

·      cookie ‘max-age=’ – sets max age in seconds

·      cookie ‘path=/’ – any page in the root can read cookies

·      cookie ‘domain=’ – sets the domain that can read the cookies

·      cookie ‘secure’ – ensures transfer of info over https network

·      cookie ‘expires=Thu, 01 Jan 1970 00:00:01 GMT' – removes a cookie by making it expire in the past

·      document.cookie = "cookie=;expires=" + new Date(0).toUTCString() – removes a cookie by setting the expiration date as now – CONFIRMED

·       

·      use cookie libraries like Cookies.js or jsCookie

·      window.setTimeout() – calls a function after a number of milliseconds. Returns an Integer which is the timeout ID

·      window.clearTimeout() – provided the right timeout ID, cancels a timeout

·      window.setInterval() – sets a timeout at a recurring interval

·      window.clearInterval() – cancels an intervaled timeout

 

The Content Template Element

 

·      <template> – holds html that renders on command from javascript

 

template filling example:

if ('content' in document.createElement('template')) {

    var template = document.querySelector('');

    var clone = template.content.cloneNode(true);

    var div = clone.querySelector("div");

    div.textContent = "";

    tbody.appendChild(clone);

}
