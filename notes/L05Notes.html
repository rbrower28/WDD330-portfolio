AJAX

 

·      AJAX is a way to run several processes at once on a webpage. It turns static pages into dynamically running ones.

·      Without AJAX, JavaScript files would only be able to communicate with the data given by the original server response.

·      With AJAX, a JS file can specifically request tidbits like .json information right in the moment without another general page server request.

·      Two options for testing with AJAX are to set up your own backend server or use a public site that supports cross-origin resource sharing (without this, you can only take resources from the same server)

·      CORS is a way to dictate which servers can be used so some protection from unwanted .js code is still there.

·      Ajax lets you refresh parts of your page without reloading the entire thing. So, if your request lags or has an issue, the rest of your webpage still works! It doesn’t freeze the whole thing. It used to be XML based but with JSON compatibility that became the most common use (AJAJ).

·      The fetch() method of the Fetch API allows asynchronous data requests.

·      looks like: fetch(‘link’).then(methods).catch(error methods)

·      you can check with the response.ok property to see if the fetch was successful

·      other response properties: headers, url, redirected, type

·      response.redirect(url) – allows page redirection – currently unsupported :(

·      response.text() – good for debugging

·      response.blob() – returns raw data of the fetched file

·      response.json() – neatly lines up the .json response into a JS object

·      HTTP requests – GET, POST, PUT, PATCH, DELETE

 

Example of fetch parameters (you can custom set):

fetch('https://example.com/data', {

    method: 'GET',

    mode: 'cors',

    redirect: 'follow',

    cache: 'no-cache'

})

 

·      The Headers() object stores more information like file type and cookies.

·      Headers .has() .get() .set() .append() .delete() .keys() .values() .entries() – headers methods

 

Textbook application:

const url = 'https:example.com/data';

const headers = new Headers({ 'Content-Type': 'text/plain', 'Accept-Charset' : 'utf-8', 'Accept-Encoding':'gzip,deflate' })

 

const request = (url,{

    headers: headers

})

 

fetch(request)

.then( function(response) {

    if(response.ok) {

        return response;

    }

    throw Error(response.statusText);

})

.then( response => // do something with response )

.catch( error => console.log('There was an error!') )

 

·      AJAX libraries like jQuery are great to use ajax while keeping it up to date. Ajax usage might change but people work hard to keep all library methods up to date indefinitely.
