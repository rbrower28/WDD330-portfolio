Forms

 

·      <form> elements are sent from the browser to the backend server and are usually handled by backend languages like PHP.

·      It can also be handled without bothering the backend server by using frontend JavaScript.

·      const form = document.forms – returns a list of all the forms on an html page.

·      form.search – allows you to search inside a form.

·      form.elements – returns elements in the style [input, button].

·      form.searchInput – lets you search within the inputs.

·      form.submit() – calls the submit event, does the same as <button type=“submit”> or <input type=“submit”>.

·      form.reset() – resets all form values to the initial setting. Warning: leads to poor usability and can be easy to cause setbacks for users.

·      form.action() – lets you set the action attribute of the form (the redirect url).

·      You can add event listeners for actions on form elements.

·      Event listeners can trigger code at the submittal of a form.

·      event.preventDefault() – stops a form from actually sending information (for tests).

·      input.value – returns the value of a selected input.

·      Note: HTML placeholder text won’t ever be returned as input.value.

·      Form Controls: <input>, <select>, <textarea>, <button>

·      Input Fields: text, password, checkbox, radio, hidden, file, number, tel, color, etc.

·      The best form validation is the browser, JavaScript, and the server all working together.

 

Using FormData Objects Effectively

 

·      The FormData Object can be used to parse forms to easily read and manipulate data.

·      This saves you from having to search manually for each form element separated from the form as a whole.

·      To properly use the FormData Object, you need to make sure that each element has a “name=” property. Otherwise, none of the information can be accessed this way.

·      using preventDefault() lets you manipulate the data with JavaScript without reloading the page, and without a server request.

 

 

 

 

Code sample to retrieve form data:

document.getElementById(“my-form”)

.addEventListener(‘submit’, handleForm);

 

function handleForm(ev) {
     ev.preventDefault();

     let form = ev.target;

     let fd = new FormData(form);

}

 

 

Client-Side Form Validation

 

·      You need to make sure forms are all valid before you send them to the server.

·      Other form validation might be too late for the user to correct, but client-side validation warns the user in the moment that the form info for a certain input is invalid.

·      Not validating user input is one of the easiest ways to not only crash the site but the servers as well. Do better.

·      Built-in server validation:  required, minlength, maxlength, min, max, type, pattern

·      Built-in CSS selectors for input elements – :valid, :invalid

·      Use regular expressions (regex)

 

 

Regex basic rules:

·      minlength, maxlength – sets limits to character amount.

·      min, max – sets limits to number entries.

·      The Constraint Validation API lets you set rules from JavaScript.

·      <form novalidate> – turns off any browser side validation (if you want completely custom)

·      To validate without an API, you need to predict the types of user errors and account for each and every one in your troubleshooting.
