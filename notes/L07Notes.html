Further Functions

 

·      Functions have properties and methods just like other data types.

·      Some function methods and properties include:

o   function.length – returns the number of parameters

o   function.call( { param: value } ) – calls a function and fills the parameters in a dictionary type style.

o   function.call(null, value) – uses the call method without dict, must put null as the first value.

o   function.apply(null, [value]) – same as call but parameters need to be in a list.

·      You can reference given parameters with this.paramName

·      function(param=“test”) – sets the default to “test” if none is provided

·      You can create your own properties for any function like .description = “desc” and it will preserve the value as an attribute.

·      funcName.cache – used inside the function, caches the same value so you can modify it without setting everything to a new variable. Great for high labor functions.

·      Immediately Invoked Functions are called in the moment they are defined.

 

These look like:

( function () {

// stuff

} ) ();

 

·      Using the variable name temp lets you redefine variables that you won’t need in the long run and keeps you from messing things up with new variable names farther down in the code.

·      IIF’s can be used to set up initial code snippets that will never be needed again, and they won’t be confused for anything outside the scope of the immediate function.

·      They can also be used to ‘use strict’; on a chunk of code without affecting other code that you may not have written. Really useful with scope.

·      Init-time branching lets us write different versions of the same function depending on variables like changes in browser compatibility.

·      if (…) f = function() { a } else (…) f = function() { b }

·      Recursivity – a function called within itself.

·      Callbacks – a function passed as a parameter and used inside another function.

·      We can use callbacks to modify any other function we have.

 

 

 

Here is a good example of this:

function wait(message, callback, seconds){

    setTimeout(callback,seconds * 1000);

    console.log(message);

}

 

·      The previous function doesn’t actually do anything for the code, but it can add to any other related function, and can be really good for debugging and running several things at the same time.

·      “Callback hell” is when you nest so many callbacks that the code is unrecognizable.

·      Promises represent whatever future result the callback function will be.

·      A promise can either return Resolved or Rejected, on whether the code runs perfectly or not.

·      Promises can be created with preset outcomes.

 

Here is a good Promise() example:

const promise = new Promise( (resolve, reject) => {

    if (success) {

        resolve(value);

    } else {

        reject(error);

    }

});

 

·      promise.then() – lets you do things with whatever a successful promise returns.

·      You can use .then to chain together as many promises as you want.

·      async/await functions will call for a promise and won’t run anything after await until the promise is resolved.

·      Using several awaits will replace a huge .then() chain and simplify multiple promises.

·      return function(){ – inside a function, returns a function for later use.

·      A closure is a reference to a variable defined in a function but that we want to use later.

·      In a counter function, the variable i is remembered and not reset when the function resets.

·      A generator (made using function* ) creates a function but doesn’t let it run completely.

·      They inherit the method .next(), which lets the function continue from the previous yield to the next yield.

·      Yield is like return but the value is stored in the function to be used again .

·      JavaScript can program with a functional, object-oriented, or procedural basis. This is referred to as a multi-paradigm language.

·      A Pure function is the result of independence and consistency.

·      Higher-Order functions deal with smaller functions, whether it be in the parameters or what is returned.

·      Currying is when a function is smart enough to divide and return smaller functions based on whatever parameters were missing. This makes finishing the function with the rest of the parameters less work for the computer.

 

CSS Transforms and Transitions

 

·      CSS Transform lets you translate, rotate, scale, or skew any html elements.

·      transform: translate(x, y); – moves the element on the screen.

·      translateX(), translateY() – moves on just x or y axis.

·      warning: transforms don’t apply to inline objects.. change them to inline-block first.

·      scale(x, y), scaleX(), scaleY() – edits element size.

·      Scaling doesn’t bump around other objects on the screen.

·      rotate(xdeg) - sets new rotation.

·      skew(xdeg, ydeg) – skews from direct view by degrees.

·      transform-origin: – resets the origin point from which the transforms happen.

·      Be careful about ordering! the same transformations in different order could have some drastically different results.

 

·      CSS Transition lets you set a movement that is repeated smoothly.

·      transition-property: – defines which properties change on a transition.

·      transition-duration: – sets duration, .5s is a half a second.

·      transition-timing-function: – more control over transition speed in certain parts.

·      transition-delay: – sets a timer delay for the transition.

·      transition: – shorthand for all the above transition lines.

 

·      CSS Animations let you control movement on a frame-by-frame basis.

·      Animations let us use many keyframes to define points in the animation. Unlike transforms that only have a start and finish, animations let you have fixed points in between.

·      @keyframes myAnimation { /* content here */} – an example.

·      animation-name: – lets you link it to an element.

·      animation-duration: – sets how long.

·      animation-timing-function: – support for different timing in different parts.

·      animation-iteration-count: – sets how many times the animation will loop.

·      animation-delay: – sets the delay before starting.

·      animation-fill-mode: – changes the effects before and after the animation.

·      animation: – animation shorthand.
